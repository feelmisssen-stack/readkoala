# 도란서재 디자인·서체 되돌리기 가이드

디자인을 바꿨다가 **지금의 색·서체·버튼 스타일**로 다시 돌리고 싶을 때 참고하는 문서입니다.

---

## 요약: 디자인만 되돌릴 수 있나요?

**네, 가능합니다.**

기능(로그인, API, DB 등)은 그대로 두고 **디자인 관련 파일만** 예전 버전으로 가져올 수 있습니다. Git으로 특정 파일만 선택해서 복구하면 됩니다.

## 방법 1: Git으로 디자인 파일만 되돌리기 (추천)

### 1단계: 어떤 파일이 바뀌었는지 확인

```bash
git diff design.css src/app/globals.css tailwind.config.ts src/app/layout.tsx
```

### 2단계: 기준 커밋에서 디자인 파일만 복구

**태그를 붙여 둔 경우:**

```bash
git checkout design-baseline-2026-06-26 -- design.css src/app/globals.css tailwind.config.ts src/app/layout.tsx src/lib/icon-styles.ts
```

**커밋 해시를 쓰는 경우:**

```bash
git checkout 49b33c7 -- design.css src/app/globals.css tailwind.config.ts src/app/layout.tsx src/lib/icon-styles.ts
```

### 3단계: 확인 후 저장

```bash
git status
git diff --staged
npm run dev
```

화면이 원하는 모양이면:

```bash
git add design.css src/app/globals.css tailwind.config.ts src/app/layout.tsx src/lib/icon-styles.ts
git commit -m "디자인·서체만 기준 버전으로 복구"
```

> `git checkout <커밋> -- <파일>` 은 **선택한 파일만** 예전 내용으로 바꿉니다.  
> 다른 기능 코드는 건드리지 않습니다.

## 방법 2: Cursor(에이전트)에게 요청할 때 쓸 문장

아래를 **그대로 복사**해서 붙여 넣으면 됩니다.

### 전체 디자인·서체 복구

```
design.css, globals.css, tailwind.config.ts, layout.tsx, icon-styles.ts 를
커밋 49b33c7 (또는 태그 design-baseline-2026-06-26) 기준으로 복구해줘.
기능 코드와 data/db.json 은 건드리지 마.
```

### 색상만 되돌리기

```
design.css 와 globals.css 의 색상·koala-* 클래스만
49b33c7 버전으로 되돌려줘. 다른 파일은 수정하지 마.
```

### 서체만 되돌리기

```
서체를 지금 도란서재 기준(Noto Serif KR, font-reading)으로 되돌려줘.
layout.tsx, design.css, tailwind.config.ts, globals.css 만 확인해줘.
```

### 특정 컴포넌트 UI만 되돌리기

```
KoalaGrowthCard.tsx 와 FriendFeedMosaic.tsx 의 스타일만
49b33c7 커밋 기준으로 복구해줘. 로직은 유지해.
```

## 방법 3: 브랜치로 실험했다가 버리기

디자인 실험을 **별도 브랜치**에서 할 때:

```bash
git checkout -b design-experiment
# … 디자인 수정 …
```

마음에 안 들면 `main`으로 돌아가면 됩니다.

```bash
git checkout main
```

브랜치만 버리기:

```bash
git branch -D design-experiment
```

## 자주 하는 실수

| 상황 | 주의 |
|------|------|
| `git reset --hard` | **전체**가 예전으로 돌아가서, 그동안 만든 기능도 사라질 수 있음 |
| `data/db.json`까지 checkout | 회원·책 데이터가 예전으로 돌아감 → **하지 말 것** |
| 컴포넌트 안의 `className`만 수백 곳 수정 | 나중에 되돌리기 어려움 → 가능하면 `design.css` / `globals.css` 중심으로 바꾸기 |
